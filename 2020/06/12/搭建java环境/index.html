搭建java环境

java
  • basics

了解配置JDK和JRE

JRE(Java Runtime Environment Java运行环境)

​ Jre包括Java虚拟机和Java程序所需的核心类库 (Jre = Jvm + 类库)

​ 如果要运行一个开发好的Java程序，计算机中只需安装Jre即可

JDK(Java Development kit Java开发工具包)

​ Jdk提供给Java开发人员使用

​ Jdk其中包含了Java的开发工具，也包含了Jre(Jdk = jre + 开发工具)

​ 安装好了Jdk，就不需要在单独安装Jre了

​ 其中开发工具举例：编译工具(javac.exe)、打包工具(jar.exe)…

JDK的下载

​ 下载网址：http://www.oracle.com/technetwork/java/javase/downloads/index.html

安装JDK

此处示例安装的是Java 8(1.8)版本的JDK，该版本也是目前使用最多的版本

  1. 下载好JDK后，双击已经下载完毕的安装文件，开始安装

  2. 选择JDK的安装目录，建议使用默认路径，放在纯英文路径下

  3. 点击下一步，如果出现下图的警告信息，直接点击 “确定” 继续

  4. 点击下一步，弹出一个窗口，这是来选择安装jre的路径的，仍然使用默认路径

  5. 安装成功

配置环境变量的三种方式

在完成上述安装JDK操作后，就可以开始配置环境变量了

那么为什么要配置环境变量呢？

必须配置环境变量才能使用Java开发环境

在win10下，只需要配置环境变量Path(可以使系统在任何路径下都能识别Java命令)即可

所以为了能在我们电脑的任何路径下都可以使用Java命令我们需要在系统环境变量中进行相应的配置

如果你是一个特特特勤快的人可以选择每次都切到JDK安装所在的路径中去运行Java命令(显然你不是)

系统环境变量

这里友情提供如何找到环境变量配置的位置

右击我的电脑 – 选择属性 – 高级 – 右下角点击环境变量

注意：下面所有的操作都是针对于 系统变量 进行

一劳永逸

所谓一劳永逸呢，就是做一次配置就可以一直使用了

在path中配置应用程序的路径，使用;来对不同的变量值就行分割

技巧配置

将Java程序的路径赋值给JAVA-HOME

然后在path中配置 %JAVA-HOME\bin% ,此处的 ‘%’ 用来区别其不是一个文件夹(获取一个环境变量值)

这样配置的好处是：路径或者文件名有改动的时候，只需对JAVA-HOME中的值做对应修改即可

临时配置

通过dos命令中的set命令完成

set用于查看本机所有环境变量的信息

在原有环境上配置新值：set path = 新值 ; %path%

注：该配置方式只在dos窗口有效，窗口关闭，配置消失

配置Classpath

classpath：类文件的路径，给JVM使用

在没有配置Classpath的情况下，默认为当前目录

配置了classpath后，优先去classpath配置的目录中去进行查找

可使用上述的几种方式对calsspath进行配置(自己动手，丰衣足食)

测试环境变量是否配置成功

JDK配置完成后，需要进行测试，来确认是否配置成功

  • windows + R 打开搜索框，输入cmd，回车启动命令提示符对话框
  • 在命令提示框中输入 “javac” ，按 “Enter”键，显示如下信息即说明JDK环境搭建成功
